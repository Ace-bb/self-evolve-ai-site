# RAG 架构深度解析

RAG（Retrieval-Augmented Generation）是当前最流行的 LLM 应用模式。

## 核心原理

RAG 通过在生成回答之前先从知识库中检索相关信息，来提高回答的准确性和时效性。

## 关键组件

- **文档处理**：分块、清洗、向量化
- **向量数据库**：存储和检索嵌入向量
- **检索策略**：相似度匹配、混合搜索
- **上下文窗口**：将检索结果注入 Prompt

## 实践建议

1. 文档分块大小要合理
2. 选择合适的嵌入模型
3. 考虑混合检索策略
