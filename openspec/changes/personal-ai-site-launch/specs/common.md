# Spec: 通用页面与设计系统

## Overview
网站整体架构、布局、设计系统、导航、SEO、性能等非模块特定的需求。

## Requirements

### REQ-22: 全局布局
- **描述**: 统一的页面布局
- **验收标准**:
  - Header: Logo + 四模块导航 + 搜索框 + 主题切换
  - Footer: 版权信息、社交链接、站点地图
  - 响应式：桌面/平板/手机三端适配
  - 移动端汉堡菜单

### REQ-23: 设计系统
- **描述**: 统一的视觉设计语言
- **验收标准**:
  - 使用 TailwindCSS 作为基础
  - 定义 Design Token（颜色、字体、间距、圆角）
  - 深色/浅色主题
  - 统一的组件库（按钮、卡片、标签、输入框等）
  - 中文优化字体配置

### REQ-24: 首页
- **描述**: 网站首页展示
- **验收标准**:
  - Hero 区域：个人介绍 + 一句话定位
  - 四大模块入口卡片
  - 最新文章预览（3 篇）
  - AI 工具箱精选推荐
  - 联系方式 / CTA 区域

### REQ-25: 关于页面
- **描述**: 个人介绍页面
- **验收标准**:
  - 个人简介
  - 技术栈展示
  - 时间线 / 经历
  - 社交链接

### REQ-26: 性能
- **描述**: 页面加载性能优化
- **验收标准**:
  - Lighthouse 性能评分 ≥ 90
  - 首屏加载 < 1.5s
  - 图片自动优化（WebP、懒加载）
  - 代码分割，按需加载

### REQ-27: 部署
- **描述**: 持续部署方案
- **验收标准**:
  - 支持 Vercel 一键部署
  - GitHub Actions CI/CD
  - push 到 main 自动构建部署
  - 支持自定义域名
